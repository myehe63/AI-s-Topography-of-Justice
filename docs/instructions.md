git checkout -b refactor-pipeline로 브랜치 만들고, 레포 루트의 pipeline_refactor_spec.md를 읽어봐. 이 스펙대로 지금 있는 코드를 리팩토링할 거야.  
코드는 아직 짜지 마. 먼저 이것부터 해줘:  
1. 레포 전체 파일 트리를 출력 (ls -R 또는 find)  
2. 문서 6번 매핑표의 '기존 파일' 칸이 실제 파일과 맞는지 하나씩 대조. 안 맞으면 어느 줄이 틀렸고 실제 파일은 뭔지 리포트하고, **매핑표를 어떻게 고쳐야 하는지 제안**해줘.  
3. 특히 "v1 삭제" 대상이 실제로 어느 파일인지 특정해줘. **삭제는 아직 하지 마.**  
이 리포트에 내 확인을 받은 다음에, 단계별로 나한테 확인받으면서 진행해. **파일 삭제는 다른 이관이 다 끝나고 결과 검증까지 된 맨 마지막에** 해줘.
